# coral-sensor-node

サンゴ礁の環境データを測るための、M5Stack ATOM S3（ESP32-S3）用センサーノードのスケッチ集。修士研究で使用している。

## 目的・背景

サンゴ礁の状態調査は、ダイバーによる目視と写真撮影が主流である。潜水作業は身体的な負担が大きく、評価に個人差も出る。衛星観測は広域の熱ストレスを捉えられるが、同一海域内の局所的な水温・光量・水質までは分からない。

そこで、現場で局所的な環境データを取得することを目的に、水中で複数の環境項目を同時に測るセンサーノードを自作している。このリポジトリは、そのマイコン側（ATOM S3）のコードをまとめたものである。

測る項目は、環境省『サンゴ礁生態系保全行動計画 2022-2030』および Coral Reef Alliance の推奨パラメータのうち、現場センサーで取得できるものから選んだ（水温・pH・塩分・溶存酸素・光量・深度）。

**現在の段階**: 対象6項目のうち4項目（光量・水温・深度・塩分）は単体で実測による動作確認を終え、1つのマイコンに統合済み。pH と溶存酸素は部品の問題で未確認。データはシリアル出力のみで、ロギング・無線送信・GPS 連携は未実装。

## 構成

### ハードウェア

マイコンは M5Stack ATOM S3（ESP32-S3）。I²C バスは G5=SCL、G6=SDA。

| 項目 | センサー | 型番 | 接続 |
|---|---|---|---|
| 光量 | Adafruit TSL2591 | — | I²C (0x29) |
| 深度・圧力 | Blue Robotics Bar02 | MS5837-02BA | I²C (0x76) |
| 水温 | DFRobot DS18B20 防水温度センサー | DFR0198 | 1-Wire (G8) |
| 塩分（EC） | DFRobot Gravity EC センサー K=10 | DFR0300-H | ADS1115 A0 |
| pH | DFRobot Gravity pH センサー V2 | SEN0161-V2 | ADS1115 A1 |
| 外部ADC | Adafruit ADS1115（16bit） | — | I²C (0x48) |
| 溶存酸素 | DFRobot 蛍光式DOセンサー | SEN0681 | 未接続（RS485） |

アナログセンサー（pH・塩分）は、ATOM S3 内蔵ADCのノイズを避けるため、すべて ADS1115 経由で読んでいる。DS18B20 は外付け 4.7kΩ でプルアップしている。

## 実装で対応したこと

`sensor_ec/sensor_ec.ino` で自分で判断・実装した部分。借用部分との区別のため、ライブラリ任せの箇所は下の「ライブラリに任せている部分」に分けた。

- **DS18B20 の読み取り失敗対策**: 断線時に `-127℃` が返るため、「読む → 範囲チェック → 使う」の順にしている。異常値のときは前回値を据え置き、表示に `(古い値)` を付ける。これをしないと、異常値がそのまま EC の温度補正や EEPROM の校正値に流れ込む。
- **EC の温度補正に使う水温の選択**: 電気伝導率は水温で値が変わる。Bar02 にも内蔵温度センサーがあるが、水温として使うのは DS18B20 の実測値にし、Bar02 の温度は参考表示に回している。
- **アナログセンサーを外部ADC経由にした判断**: ATOM S3 内蔵ADCのノイズを避けるため、pH・塩分は ADS1115（16bit）経由で読む構成にした。
- **ESP32 での EEPROM 初期化**: `EEPROM.begin(32)` を追加している。ESP32 の EEPROM はフラッシュ上のエミュレーションであり、`begin(size)` で領域を確保しないとライブラリ側の読み書きが成立しない。DFRobot_EC10 付属サンプルにはこの呼び出しがない。
- **I²C バスの共存**: Bar02 (0x76)、TSL2591 (0x29)、ADS1115 (0x48) はアドレスが異なるため、同一バスに接続している。

### ライブラリに任せている部分

- **EC 校正コマンド**（`ENTEREC` → `CALEC` → `EXITEC`）の受信・解析、12.88 mS/cm 標準液の自動認識、K値の EEPROM 保存は、すべて `DFRobot_EC10` ライブラリの機能である。`sensor_ec.ino` 側は `ec.calibration(voltage_mV, waterTemp)` を呼んでいるだけ。
- **lux の算出**は `Adafruit_TSL2591` の `calculateLux()`、**深度の算出**は `MS5837` ライブラリによる。

## 使用技術

- C++ / Arduino IDE
- M5Stack ATOM S3（ESP32-S3）
- I²C / 1-Wire / アナログ入力（ADS1115 経由）

依存ライブラリ:

- `Adafruit_TSL2591` および `Adafruit_Sensor`
- `MS5837`（Blue Robotics Bar02 用）
- `Adafruit_ADS1X15`
- `DFRobot_EC10`
- `OneWire` および `DallasTemperature`

## 実行方法

1. Arduino IDE に ESP32 ボード（M5Stack ATOM S3）を追加する
2. 上記の依存ライブラリを入れる
3. 配線する（I²C: SDA=G6 / SCL=G5、DS18B20: DATA=G8 に 4.7kΩ プルアップ）
4. `sensor_ec/sensor_ec.ino` を書き込む
5. シリアルモニタを 115200 bps で開く

出力形式:

```
光量: <lux> Lux  DS18B20水温: <℃> C  Bar02温度: <℃> C  深度: <m> m  塩分電圧: <V> V  EC: <mS/cm> mS/cm  pH電圧: <V> V
```

淡水で試す場合は `setFluidDensity(997)`、海水では `1029` に変更する。

## 動作確認の結果（2026年7月時点）

対象6項目のうち4項目を実測で確認した。

| センサー | 状態 | 確認内容 |
|---|---|---|
| 光量 TSL2591 | 確認済み | 晴天の直射日光下（TSL2591 単独測定）で 64,143〜65,004 Lux を出力し、頭打ちにならず変化した。別日の曇天下（21,700〜30,600 Lux）で照度計 LX-1010B と並べたところ、**LX-1010B ≈ TSL2591 × 約1.85** の一定比率で追従した。スペクトル感度の違い（LX-1010B は可視光、TSL2591 は可視光+IR補正）による系統差で、補正係数で換算できる。最高照度域での照度計照合は未実施 |
| 水温 DS18B20 | 確認済み | 温度計と比較し、常温水・氷水の両方で一貫して約0.8〜1.0℃低い。ランダム誤差ではなく系統オフセットのため、補正で一致させられる |
| 深度 Bar02 | 確認済み | 14階建て（高さ約61m）の建物で、1階が14階より **+7.24 mbar 高い**ことを測定（各地点のブレは 0.05〜0.06 mbar）。標高が上がると気圧が下がるという向きと一致し、量も気圧の目安（標高10mで約1 mbar）から見て妥当なオーダーだった。水中テストは防水ケース後 |
| 塩分 DFR0300-H | 確認済み | 校正液（12.88 mS/cm）で 0.38V。校正後、3.5%塩水（海水相当）で約42 mS/cm を示し、海水（45〜50 mS/cm）に近い値を確認。水道水・イオン除去水では無反応で、これは測定範囲 10〜100 mS/cm の仕様どおり |
| pH SEN0161-V2 | 未確認 | pHアンプ基板のアナログ出力端子をテスターで直接測ったところ、空気中・水道水・pH9.18標準液のいずれでも約1.82V で固定し変化しなかった。センサー側の問題と判断して再発注中 |
| 溶存酸素 SEN0681 | 未確認 | RS485（Modbus-RTU）のため、変換アダプタ DFR0845 の到着待ち。コードは未実装 |

### 光量の「飽和しなかった」について

晴天下で 64,143〜65,004 Lux が出て頭打ちにならなかったことは実測で確認したが、**コード上で飽和していないことを確認したわけではない**。`Adafruit_TSL2591` のオーバーフロー判定は生カウント `ch0` または `ch1` が `0xFFFF` のときに `-1` を返す実装だが、積分時間 100ms ではフルスケールが `0xFFFF` に届かないため、この判定は発火しない。また、よく引かれる 88,000 Lux はデータシート上のダイナミックレンジ上限であり、GAIN_LOW / 100ms という現在の設定での飽和点ではない。飽和を確実に見るには、生カウント（`full` / `ir`）と `calculateLux()` の戻り値 `-1` を確認する必要がある。

## 今後の予定

- pH センサー到着後の動作確認と、2点校正（6.86 / 9.18）による pH 値換算の実装
- 溶存酸素センサー（SEN0681）の Modbus-RTU 対応
- EC 校正値が電源断で消える問題への対策。`DFRobot_EC10` は `EEPROM.write()` のみで `EEPROM.commit()` を呼んでいない。ESP32 の EEPROM はフラッシュ上のエミュレーションのため、commit しないと RAM 上のバッファに載るだけで保存されない。AVR 前提のライブラリが ESP32 に移植されていないことが原因と考えられるので、K値をコードに直接埋め込むか、commit を補う方法を検討する
- 飽和判定の実装。生カウント（`full` / `ir`）の表示と `calculateLux()` の `-1` 判定を入れる
- DS18B20 の遮光の要否判断。晴天の屋外テスト中、直射日光下で約16秒のうちに 36.19→37.06℃（+0.87℃）の上昇を観測した。長時間の光曝露による破損の懸念が指摘されており、浅海では水中でも直射光が届くため、水中テストまでに判断する
- 防水ケース作成後の水中テスト
- GPS（水面ブイ側）との時刻同期によるデータ統合
- サンプリング間隔の短縮。1サイクルには `delay(1000)` に加え、DS18B20 の変換待ち（12bit で約750ms）と TSL2591 の積分待ち（ライブラリ実装で最低120ms）、Bar02 と ADS1115 の読み取りがブロッキングで積み上がる（1サイクルの実測はしていない）。移動しながら測る場合、この間隔がそのまま空間分解能に効くため、DS18B20 の分解能を下げる（9bit で約94ms）か非同期読みにするかを、精度とのトレードオフで決める
